# POS Automation Test — ระบุปลายทาง

ทดสอบการกดไปกลับระหว่างหน้า **ระบุปลายทาง** และ **เลือกบริการ** แบบวนซ้ำ

---

## ความต้องการ

- Python 3.x
- โปรแกรม **Riposte POS Application** รันอยู่บนเครื่อง Windows
- ติดตั้ง pywinauto

```bash
pip install pywinauto
```

---

## วิธีใช้งาน

### 1. เปิด POS และ Navigate ไปที่หน้าระบุปลายทาง

เปิดโปรแกรม Riposte POS แล้วกดเข้าไปที่หน้า **ระบุปลายทาง** (หน้ากรอกรหัสไปรษณีย์) ให้เรียบร้อยก่อน

### 2. เปิด Terminal แล้วรัน

```bash
cd C:\Users\THPCORE\pos-script
py test_back_forth_postal.py
```

### 3. หยุดการทดสอบ

กด **Ctrl+C** ใน Terminal เมื่อต้องการหยุด โปรแกรมจะแสดงสรุปผลให้อัตโนมัติ

---

## การตั้งค่า

แก้ไขค่าได้ที่ด้านบนของไฟล์ `test_back_forth_postal.py`

| ตัวแปร | ค่า default | ความหมาย |
|---|---|---|
| `REPEAT_COUNT` | `None` | จำนวนรอบ — `None` = วนไปเรื่อยๆ, ใส่ตัวเลข เช่น `10` = หยุดเองที่ 10 รอบ |
| `POSTAL_CODE` | `"10210"` | รหัสไปรษณีย์ที่ใช้กรอกในแต่ละรอบ |
| `STEP_DELAY` | `1.0` | วินาทีที่รอระหว่างแต่ละ step |
| `PAGE_TIMEOUT` | `15` | วินาทีรอสูงสุดให้หน้าโหลด (เพิ่มถ้าระบบช้า) |
| `PAGE_POLL` | `0.5` | ความถี่ในการตรวจสอบสถานะหน้า (วินาที) |

ตัวอย่าง — หยุดเองที่ 20 รอบ:
```python
REPEAT_COUNT  = 20
```

## สถานะผลลัพธ์

| สถานะ | ความหมาย |
|---|---|
| `PASS` | กดไปกลับสำเร็จครบรอบ |
| `FAIL` | เกิดข้อผิดพลาด เช่น หา edit box ไม่เจอ, หน้าไม่โหลด |
| `SKIP` | ไม่ได้อยู่ที่หน้าระบุปลายทางตอนเริ่มรอบ |
